    Introduction

    Rainhut API

    Welcome to the Rainhut API. Access the API endpoints to create digital media: books, cards, flyers, and more.

    Start building your own content now. Contract Rainhut at support@rainhut.com

    We have language bindings in JavaScript. As you browse this document, view code examples in the dark area to the right.

    Visit the corresponding JavaScript Github repository here:

    https://github.com/rainhut2/rainhutapi

    Authentication

    To authorize, use this code:

    
var rainhutapi = require("rainhutapi")
var api = new rainhutapi(privateKey, publicKey);

    Rainhut uses private and public token keys to allow access to the API. You can register by contacting us at support@rainhut.com

    You must replace 'privateKey', 'publicKey' with your personal private and public keys. Checkout the javascript api for example on how to authenticate your own app.

    Rainhut Setup

    For every call you will need to pass a setup object.

    Parameter Default Description
    paddingLeft 0 Will set padding on page left
    paddingRight 0 Will set padding on page right
    paddingTop 0 Will set padding on page top
    paddingBottom 0 Will set padding on page bottom
    headerHeight 50 Header height
    footerHeight 50 Footer height
    columnSpacing 10 How much space between columns
    entrySpacing 10 How much space between each item
    width 1000 Document width. Including padding
    height 800 Document height. Including padding
    questionStyle defaultStyle Style for entry question
    answerStyle defaultStyle Style for entry answer
    footerStyle defaultStyle Style for footer
    headerStyle defaultStyle Style for header
    dateStyle defaultStyle Style for date
    maxFontSize 50 Max font size to increase. In pixels.
    cropImage false Will crop images to fit into layout if set to true.
    imageClass '' Any custom image effects that have been setup for your layouts. Possible options for Qeepsake "imageBorderShadow1", will add a show box around image.
    minimumDpi 90 Sets minimum dpi for images to scale
    highResDpi 150 Sets what a highres dpi for image is
    birthday '2017-22-10' Birthday of given journal entry book
    backgroundImage "" Will add a background image to the pages. Can take a local file or a url to an image. Preferably svg. "./../img/swish.svg" is current local background.
    rotateAmount 0 Will rotate image randomly by degrees + or -. Passing 0 will not rotate image.
    minTextFullPage 650 Rule to specify what a full page text will look like. 650 characters is the break amount for a full page entry.
    boldFirstAnswer true Will bold first sentence in answer if not question is available.
    randomForEqual false Will randomize options for equal, best layout choices. If false will alternate between best choices.

    Styles

    textColor 0 Will set padding on page left
    fontFamily 0 Will set padding on page right
    size 1.0 Font size on relative scale set to 18px. Font can increase up to given maxFontSize
    weight normal Font weight. Can be normal, italic, bold, bold italic

    API Endpoints

    Create Book

    HTTP Post

    Post https://artapi2.rainhut.com/api/create2

    Body Parameters

    Parameter Default Description
    setup defaultSetup See Setup Params
    entries [] Entries object
    needsImageSizes true Setting to false will not check for setting image size. No matter what is passed in imageWidth and imageHeight.
    Image size will be detected automatically. Entry imageSize is optional, but if you want to insure the image size is correct, pass from your own data. If you pass imageWidth and imageHeight then it will never be calculated if if you set the flag to true.

    Entry

    Parameter Default Description
    imageWidth 0 Image width, Int
    imageHeight 0 Image height, Int
    question '' question for entry
    answer '' answer for entry
    date '' Date for entry. UTC format
    footer '' Footer, pass Generate_AGE to calculate age based on the last entry on page and the birday passed in setup.
    header '' Header string
    isHeader false Will choose from Header layouts for the page.
    isSingleEntry false Will force this to be the only entry on a page.
    forceLayout '' Will force the layout of a given one entry. Only singe entry items will work or double image and text.

    Description

    Passing entries and setup will return a book with different layouts.

    This is the start for creating a book.

    Returns

    Parameter Type Description
    bookId String unique id for generated book
    pages Array of page returns an array of each page

    Page

    Parameter Type Description
    layout String current layout
    layoutPreview String url to image
    otherLayoutOptions Array of String other options including current option. Sorted best to least (if applicable)
    entries Array of entry Each entry for the current page

    Entry extra parameters returned.

    Parameter Type Description
    imageFirst Bool Does image layout first. Might not be applicable for given layout.
    rotateImage Int Rotates image by degrees can be negative or positive 
    const rainhutApi = require('rainhutApi');
let api = rainhutApi('privateKey', 'publicKey')

    Update Book

    HTTP Post

    Post https://artapi2.rainhut.com/api/update2

    Body Parameters

    Simple pass the book object returned from create2.

    Extra parameters

    Parameter Type Description
    layoutToBestFit Bool Will change the layout to best fit the changed content. Important when removing entries or modifying large amounts of text or change images. Defaults to false.
    needsImageSizes will default to false for this call
    Remove any pages that you are not updating. Replace the page(s) that you passed with the pages that are returned.

    Upload Book

    HTTP Post

    Post https://artapi2.rainhut.com/api/upload2

    Body Parameters

    Simple pass the book object returned from create2 or the modified update2 book

    Extra parameter.

    Webhook

    Parameter Type Description
    webhook Object Callback for the status of the upload of each page

    Webhook object

    Parameter Type Description
    url String Your callback url
    headers Object Any headers you want to pass back ie.

    Headers example

    {headers: {authentication: 'test'}}

    Returns

    Parameter Type Description
    status String success or failed
    bookId String bookId of pdf being processed
    index Int pageIndex
    url String url of pdf
    message String error message if failed

    Errors If the webhook is returning anything other than a 200 status code. It will retry up to 5 times.

    Example of webhook object

    {webhook: "urltocallbackgoeshere", headers:{authentication: "test"}}

    Webhook returns

    //success
 {"status": "success", "bookId": bookId, "index": pageIndex, "url": result}

//failed
{"status": "failed",  "bookId": bookId, "index": pageIndex, "message":  errorMessage}

    Errors

    Any json returned with "status": "error" the request has failed with the following message.

    The Rainhut API uses the following error codes:

    Error Code Meaning
    400 Bad Request -- Your request is invalid.
    401 Unauthorized -- Your API key is wrong.
    403 Forbidden -- Administrators only.
    404 Not Found -- The specified Rainhut endpoint could not be found.
    405 Method Not Allowed -- You tried to access a Rainhut with an invalid method.
    406 Not Acceptable -- You requested a format that isn't json.
    410 Gone -- The Rainhut requested has been removed from our servers.
    429 Too Many Requests -- You're requesting too many Rainhut! Slow down!
    500 Internal Server Error -- We had a problem with our server. Try again later.
    503 Service Unavailable -- We're temporarily offline for maintenance. Please try again later.
